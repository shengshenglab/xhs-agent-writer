# 常用操作模板

这个文件解决 4 个问题：
1. 纯 Agent 版怎么用
2. 不按模板也能怎么用
3. 怎么让 agent 记住你的最终文案表达
4. 换 agent 后怎么接着用

---

## 场景 1：写一篇新文案

### 在 Claude Code / Cursor / 能读文件的 agent 中

你可以结构化输入：

```text
先读取 agent/system-prompt.md、agent/style-profile.md、agent/memory-bank.md。
按里面的流程，帮我写一篇小红书文案。

主题：___
目标人群：___
内容类型：___
参考调性：___
关键词：___
```

也可以直接自由表达：

```text
先读取 agent/system-prompt.md、agent/style-profile.md、agent/memory-bank.md。
我想写一篇介绍这个工具/仓库的小红书文案。
读者是___。
我想重点表达___。
风格希望更像___。
```

默认规则：
- 推荐主标题控制在 20 字以内
- 默认输出包含标题、摘要、正文、标签
- 其中摘要不是普通概括，而是更想让人点进去的一句话介绍
- 如果我后面继续补充要求，你要继续按评分迭代更新
- 如果你能写文件，把每轮稿子保存到 `history/任务目录/`

### 在 ChatGPT / Gemini / 不能读本地文件的 agent 中

1. 复制 `agent/bootstrap.md`
2. 把 `agent/style-profile.md` 内容粘到 bootstrap 里的风格档案区域
3. 如果已有记忆库，也把 `agent/memory-bank.md` 粘进去
4. 在最后补充你的需求，可以是模板，也可以是自由表达

## 场景 2：我想到哪说到哪，agent 自己补全

直接说：

```text
我想写一篇种草向的小红书文案，介绍这个仓库。
读者是爱发小红书、也愿意借助 AI 提高文案质量的人。
我想让它更像爆款口吻，但别太油。
摘要也要更想让人点进去。
```

agent 应该：
- 自动推断缺失字段
- 生成初稿
- 打分
- 继续迭代
- 记录每轮改动

---

## 场景 3：把最终选用文案记住

如果你最后选中了某个版本，或者你自己又手动改了一版，直接这样说：

```text
这是我最终发的版本：
【粘贴最终文案】

请你：
1. 保存这篇终稿
2. 更新 memory-bank.md
3. 以后写新文案时，更贴近这个表达方式
```

如果是从当前任务里选中的版本，也可以这样说：

```text
我最后选 round 3 的版本。
请直接把当前任务目录中的最后生成稿视为最终选用版，并更新记忆。
```

agent 应该：
- 把终稿保存到当前任务目录
- 更新 `agent/memory-bank.md`
- 后续优先参考这些真人终稿的表达

---

## 场景 4：换 agent 后继续用

### 新 agent 支持读文件

把整个 `xhs-agent-writer/` 目录给它，然后说：

```text
先读取 agent/system-prompt.md、agent/style-profile.md、agent/memory-bank.md。
以后按这个框架帮我写和优化小红书文案。
默认输出标题、摘要、正文、标签。
如果我后面补充新要求，也继续按评分迭代更新。
```

### 新 agent 不支持读文件

把 `agent/bootstrap.md` 全文粘过去，再补上风格档案和记忆库内容。

## 场景 5：纯对话学习风格

当 AI 给出定稿后，你自己改了一版，回来这样说：

```text
这是 AI 定稿：
【粘贴 AI 定稿】

这是我最终发布的版本：
【粘贴人工修改后的版本】

请按 agent/program.md 的流程：
1. 做 diff 分析
2. 提取风格规则
3. 先给我确认
4. 我确认后更新 style-profile.md
5. 不管规则是否升级，都先把终稿写进 memory-bank.md
6. 如果能写文件，也把这次样本和迭代稿存到 history/ 目录
```

---

## 判断有没有“跑起来”

如果 agent 做到了下面几点，就说明框架已经生效：

- 用户不按模板填也能开始生成
- 推荐主标题会控制在 20 字以内
- 会输出 3 个标题，而不是 1 个
- 会默认输出一个更想让人点进去的摘要
- 会自己打分，而不是只给结果
- 会按最低分维度迭代，而不是整篇重写
- 用户追加新要求后，会继续纳入后续评分和迭代
- 用户提供终稿后，会保存全文并更新 `memory-bank.md`
- 后续新文案会越来越像用户本人会发的版本
- 能在 `history/` 里看到轮次记录
