# XHS Copywriter Agent Instructions

这个仓库现在是纯 agent 使用的小红书文案工作流。

## 首次进入仓库时要做什么

1. 优先读取 `agent/system-prompt.md`
2. 再读取 `agent/style-profile.md`
3. 如果存在，再读取 `agent/memory-bank.md`
4. 如果用户要看完整规则或做风格学习，再读取 `agent/program.md`
5. 如果用户想看常用对话模板，再读取 `agent/workflow.md`

## 默认任务模式

当用户让你写小红书文案时，按下面流程执行：

1. 先解析用户输入；用户可以给完整字段，也可以只说自己想表达什么
2. 优先从上下文中推断：主题、目标人群、内容类型、参考调性、关键词、附加要求
3. 只有在缺失信息会明显影响文案质量时，才追问少量关键问题
4. 生成初稿：3 个标题备选、1 个点击型摘要、正文、标签
5. 立即按 5 个维度评分：钩子力、共鸣感、信息密度、行动力、平台适配
6. 同时做硬性约束检查，尤其是：
   - 推荐主标题必须在 20 字以内
   - 默认输出必须包含标题、摘要、正文、标签
   - 摘要必须是一句更想让人点进去的话，不能只是普通概括
   - 用户临时补充的新要求必须纳入本轮检查
7. 找到最低分维度，只围绕这个维度优化
8. 最多迭代 5 轮；达到总分 >= 40/50 且无单项 < 7，且硬性约束全部满足时停止
9. 最终输出：定稿文案、迭代日志、最终评分卡、优化复盘

## 迭代稿保存

如果 agent 可以写文件，每次生成和迭代都保存到 `history/` 下的任务目录，至少包含：

- `round-01.md`、`round-02.md` ...：每轮完整文案
- `final.md`：当前 AI 定稿
- `notes.md`：每轮改了什么、为什么改

## 终稿记忆功能

如果用户后续明确提供了“最终选用版本”或“人工修改后的发布版本”，要执行两件事：

1. 记录最终使用版本
   - 如果用户是直接选用当前历史中的最后生成稿，不额外创建新文件，直接把当前任务目录中的最后生成稿视为最终选用版
   - 如果用户提供了人工修改后的发布版本，保存到 `human-final.md`
2. 更新 `agent/memory-bank.md`
   - 记录这篇终稿的主题、时间、文件路径
   - 提取这篇文案的表达习惯、句式节奏、常用词、开头方式、收尾方式

后续生成新文案前，必须同时参考：
- `agent/style-profile.md`
- `agent/memory-bank.md`

目标不是机械复读旧文案，而是让新文案更接近用户本人会说的话。

## 新要求的处理方式

用户在后续对话里新增的任何文案要求，都要视为当前任务的有效约束，例如：

- 标题长度
- 摘要点击力
- 语气偏好
- 是否要更像爆款
- 是否减少 emoji
- 是否保留某个表达

这些要求不能只临时执行一次，而要进入后续评分、迭代和定稿检查。

## 风格约束

- 生成前先使用 `agent/style-profile.md` 中的风格规则
- 再结合 `agent/memory-bank.md` 里的真人终稿记忆
- 如果这两个文件都还没有内容，就按通用小红书规则生成
- 默认使用中文输出，语气自然、口语化、适合小红书阅读

## 风格学习

当用户提供 AI 定稿和人工最终版时：

1. 做 diff 分析
2. 提取稳定的风格规则
3. 先给用户确认
4. 用户确认后，再更新 `agent/style-profile.md`
5. 与此同时，把完整终稿和样本索引写入 `agent/memory-bank.md`
6. 如果用户需要，也把这次样本和迭代稿一起存到 `history/`
